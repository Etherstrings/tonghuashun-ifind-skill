# 能力矩阵

这份文档专门给 Agent 看。

目的不是讲实现细节，而是明确回答三件事：

1. 这个 skill 当前到底能处理什么问题
2. 哪些能力同花顺 iFinD 和免费源都能拿到
3. 哪些能力只有 iFinD 有，或者当前根本没有稳定能力

## 使用规则

- 用户问题先对照本文件判断属于哪一类能力
- 如果属于“同花顺 + 免费源都有”，没有 iFinD 账号时也继续调用这个 skill
- 如果属于“只有 iFinD 有”，且用户没有 iFinD 账号，就明确告诉用户当前没有稳定公开源兜底
- 如果属于“当前没有稳定能力”，不要乱猜 payload，不要假装能做

- 常见路由没命中时，不要立刻裸写 `api-call`
- 先运行 `endpoint-list` 看当前是否已有命名接口封装
- `endpoint-call` 适合“能力已经知道、payload 也知道，但不想硬编码 endpoint 字符串”的场景

## A. 同花顺和免费源都能拿到

这些问题即使没有 iFinD 账号，也应该继续使用这个 skill：

- 个股实时行情
- 个股历史走势
- 单日开盘价 / 收盘价 / 高低点
- 大盘 / 指数快照
- 涨停数据
- A 股榜单
- 成交额榜、涨跌幅榜、换手率榜、振幅榜、量比榜

### 真实样例

`600004.SH` 在 `2026-04-21` 的免费源实测结果：

- 腾讯历史免费源：开盘 `8.88`，收盘 `8.89`
- 腾讯实时免费源：名称 `白云机场`
- 东方财富免费源：量比榜能力可用

说明：

- 当前机器上东方财富“单股快照”接口连接偶发不稳定
- 这不影响“免费源存在量比数据”这个结论
- 对 Agent 来说，应判断为“量比类能力免费源可覆盖”，而不是“没有 iFinD 就不能查”

## B. 只有 iFinD 稳定可用

这些问题当前主要依赖 iFinD：

- 基础财务指标
- 个股画像 / 主营业务
- 资金流相关问法
- 更复杂的专业数据、研报、日期序列

处理规则：

- 有 iFinD 账号：继续调用这个 skill
- 没有 iFinD 账号：明确告诉用户当前没有稳定公开源兜底

## C. 当前不要假装能做

以下问题当前不要告诉用户“已经支持”：

- `帮我找前15%市值的沪深股票`
- `筛出市值前 20% 且换手率大于 3% 的股票`
- `按多个条件做全市场选股并返回结果`

原因：

- 这类问题属于“全市场筛选 / 选股条件组合”
- 当前没有稳定自然语言路由
- 免费源也没有现成通用兜底
- 即便 iFinD 理论上能做，当前 skill 也还没把这类条件拆分成稳定 payload

对外回复应使用类似口径：

`当前 tonghuashun-ifind skill 还没有稳定覆盖这类全市场筛选能力。`

## D. Agent 决策口诀

- 先问自己：这是行情、榜单，还是基本面、画像、资金流、筛选
- 行情和榜单：没有 iFinD 账号也继续用
- 基本面、画像、资金流：没有 iFinD 账号就直接说当前没有稳定公开源
- 全市场筛选：当前不要装作能做
